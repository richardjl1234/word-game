英语单词闯关游戏开发计划书
一、项目概述

这是一个纯前端的单词闯关游戏，核心玩法为“单词雨下落匹配”。单词气泡（或词块）从屏幕上方向下飘落，玩家需要通过移动光标/点击，将下落单词与底部显示的正确释义进行匹配，匹配成功则“击中/消除”单词并得分，匹配失败或单词落地则扣血/减分。游戏包含50个难度递增的关卡，覆盖约2000个基础英语词汇。
二、核心架构建议

采用纯HTML5 + CSS3 + JavaScript（原生ES6+）技术栈，不依赖后端——所有关卡数据、单词库、音效和动画资源均前置打包到前端，孩子打开浏览器即可游玩，老师/家长也可以随时离线使用。

    HTML：构建页面骨架和核心UI元素。

    CSS：负责游戏界面风格、动画效果、关卡选择等样式表。

    JavaScript：游戏引擎逻辑，包含词库数据结构、单词下落物理引擎、碰撞检测、分数逻辑与关卡切换。

    Assets / local data：单词库JSON文件，内置2000个单词及分级难度。

三、2000词库构建与分级
3.1 词库来源建议

建议采用 “牛津2000核心词 + 中国课标基础词汇” 双轨融合的方式：

    牛津2000核心词（Oxford 3000/5000系列的基础部分） ：包含日常生活、学术及职场交流中的最常用单词，覆盖A1-B1级别的核心词汇。前2000个高频词汇可覆盖约80%的日常英文文本。

    《基础英语词汇2000》参考结构（赖世雄编著） ：全书按词汇使用频率与难易度划分为基础、进阶与流利三个层级，每单元精选约40词并以渐进式结构编排。

    新课标基础词汇（初中+高中必修） ：初中阶段要求掌握约1600-2000个基础单词，高中必修再增加500-600词。

3.2 分级策略

将2000个单词按照以下五个维度划分为 50个难度级别（Level 1至Level 50） ，每级约40个单词：
分级维度	简单级（L1–L10）	中等级（L11–L30）	较难级（L31–L50）
词长	3–5个字母	5–8个字母	8+个字母
使用频率	前500高频词	501–1200高频词	1201–2000高频词
拼写复杂度	规则拼写	常见不规则拼写	多音节、特殊发音
对应学段	小学低年级	小学高年级/初中	初中/高中基础

四、游戏核心玩法引擎
4.1 核心循环机制

采用 “下落—显示—点击匹配” 的逻辑流程：

    初始化：加载当前关卡的单词列表，顶部随机生成单词气泡，匀速向下移动。

    底部释义区：屏幕下方显示当前需要匹配的单词中文释义（或英文释义）。

    匹配判定：

        用户移动鼠标/手指点击上方的单词气泡 → 与底部当前显示的释义进行比对。

        匹配正确 → 气泡消除 → 播放“击中”音效 + 加分 → 进入下一个匹配目标。

        匹配错误 → 气泡依然存在 → 播放“失误”音效 + 扣分或扣一点生命值。

    落地判定：若某单词气泡降落到屏幕底部仍未匹配 → 播放“落地”音效 + 扣分/扣血。

    过关判定：当前关卡单词全部消除 → 显示过关动画 → 进入下一关卡或回到关卡选择界面。

4.2 渐近式难度控制

随着关卡提升自动调节以下参数以增加挑战：

    下落速度（从慢到快）——随关卡等级逐步增加。

    同时存在的单词气泡数量（从2个到6–8个）——控制屏幕密度。

    单词长度与复杂度——用词库表中难度字段作自动筛选。

    释义区的刷新频率——在较高难度下，释义的切换时间间隔缩短。

4.3 技术实现参考（Canvas路线）

碰撞检测：检测鼠标/触摸点坐标与气泡元素坐标范围是否相交。

下落动画：采用 requestAnimationFrame() 逐帧更新气泡Y轴坐标，动态修改CSS的 transform 或Canvas重新渲染。利用JavaScript动态创建单词元素，并结合CSS的绝对定位和 requestAnimationFrame 实现平滑下落动画。

物理参数设计：

    基础下落速度：约1–2px/帧（以60fps计，约60–120px/秒）。

    关卡速度系数：基础速度 × (1 + Level/50 × 最大倍率)，最高速度不超过初始的3倍。

    生成频率：每30–60帧生成一个新单词气泡。

五、AI动画预处理管线
5.1 预制动画方案

你提到“用AI预处理每个单词被正确击中时产生动画，以及落地时的不同效果，并存储到对应的JSON字段中” —— 这是一个很有前瞻性的设计。建议采用 Lottie JSON动画格式 存储：

    Lottie是一种轻量级的JSON动画格式，体积小、可缩放、跨平台，非常适合网页游戏中的资源集成。

    使用AI动画生成工具（如OmniLottie：根据文本指令生成Lottie JSON动画）进行预处理。

    击中动画（hit_animation） ：AI生成如“光芒四射”、“小动物欢呼跳跃”、“单词爆裂成字母碎片”等视觉特效。

    落地动画（fall_animation） ：AI生成如“土拨鼠摇头叹气”、“单词沉入泥土”、“表情由笑脸变哭脸”等效果。

5.2 批量预处理器流水线

用Python脚本或Node.js批量调用AI API，完成全部2000单词动画的自动预处理：

words_list.txt → AI动画API批量请求 → 返回JSON动画流 → 存入assets/animations/hit/和assets/animations/miss/文件夹 → 更新words.json词库关联路径。

5.3 小动物/拟人化处理方案

建议将每个单词气泡外部包裹一个“小动物精灵”或“拟人化头饰”，不同难度类型对应不同动物形象：
难度段	动物形象	击中后动画特效
L1–10（简单）	小兔子、小松鼠	兔子欢呼弹跳、撒花
L11–30（中等）	小猫、小狗	猫伸懒腰、狗摇尾巴
L31–50（较难）	小狐狸、小熊	狐狸露出胜利微笑、熊击掌

AI可在预处理阶段，根据单词难度，在动画模型中自动加入这些动物形象及表情变化。
六、UI/UX创意设计建议

    关卡地图选择界面：以“冒险岛”或“星河图”形式呈现50个关卡节点，每过一关点亮一颗星。

    背景变化：简单关卡是明媚花园背景，中等关卡切换到森林/古堡，高难度关卡调整为星空或海底背景。

    实时反馈区：屏幕顶部显示实时得分、连击Combo数、当前关卡进度（已消除/总词数）。

    音效系统：

        正确击中：轻快“叮”或小动物欢呼声。

        匹配错误：短促“哔哔”或动物叹气音。

        落地失败：“嗡——”低沉音效。

        过关通关：庆典号角或礼炮音效。

    孩子视觉友好：主色调为暖色系（橙黄淡蓝），圆润字体（如Comic Neue、Quicksand），按钮设计为圆角大号交互区域便于触屏。

    移动端适配：响应式设计，手机触屏一样支持手指点选单词匹配，不仅限于鼠标/光标。

7.2 关键技术点

    Canvas vs DOM元素：建议使用Canvas绘图实现批量下落与碰撞检测，性能更优；如果需要复杂的CSS交互或触摸事件响应，也可结合Canvas命中检测。

    得分与进度存储：采用浏览器的 localStorage 记录玩家的最高通关记录和当前进度。

    关卡进度控制：过关时自动从words.json中加载下一关数据。

    音效预加载：页面加载时预加载所有音效资源，避免游戏过程中的播放延迟。
